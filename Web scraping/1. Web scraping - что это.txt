Иногда появляется необходимость достать какие-то данные с сайта и у сайта не api - например курс доллара.
Web scraping - это про эту задачу - уметь вытаскивать нужную информацию

Web scraping - это процесс извлечения информации из интернета. Когда мы говорим web scraping, подразумевается автоматизация этого
процесса.

Законен ли это процесс? Да - если не сказано иного. В корне сайта есть обычно файл robot.txt - который указывает что можно брать а что нет.
Хранить же данные людей - можно только с разрешения этих самых людей.

Когда нам нужен скрапинг
1.Врема на написание скрипта меньше, чем потраченное на эту же работу без скрипта.
2.Важна скорость получения результата
3.Необходимо дождаться наступления определенного события
4.Ваши предложения?

Проблема скрапинга может быть следующая - что код может меняться и надо следить и актуализировать его, хотя внешне сайт
может выглядеть прежне.
Также сайты - могут быть с динамическим контентом и если мы его прогрузим у себя то динамический контент мы не увидим - так как мы
прогрузили только часть до доп контента.
