Pull request - это запрос на слияние. Удобный способ предложить свои изменения в командный проект в формате, который
предлагает код-ревью.
Подробнее о применении на Рисунке 2.

Далее редактируем файл и создаём новую ветку -> в разделе 
pull request на git hub

Мы переходим в 
New pull request

Затем мы выбираем какую ветку мы предлагаем как изменения и в какую ветку перелить.
Нажимаем 
Create pull request

Далее мы заполняем название, а также описание 
В описание через символ #
Можно обратиться к issue

Справа также можно настроить для кого будет предназначен этот pull request
И кто должен будет выполнить данную работу и тд
Это мы рассмотрели создание 
pull request 

Если у нас есть права доступа.
Теперь рассмотрим, что делать если прав нету.
Первое, что надо сделать - это создать fork.

После создания fork можно вновь предложить изменения, которые хотим внести в проект.
В этом случае можно не создавать новую ветку, а заливать в main, так как он попадёт именно в вашу копию fork.
Далее в самом репозитории есть два способа предложить pull request 
1. Нажать кнопку contribute -> open pull request
2. Перейти в проект, куда хотим предложить изменения и нажать pull request, но там уже нам вместо выбора веток, которые
хотим слить, мы жмём compare across fork
И затем нужно будет выбрать свой fork

Далее доводим процедуру создания pull request до конца.
