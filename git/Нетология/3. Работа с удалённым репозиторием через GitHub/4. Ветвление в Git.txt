Тут мы научимся создавать ветки, публиковать их и сливать.
Чтоб узнать, что такое ветка и для чего нужна можно посмотреть в Рисунок 4 и Рисунок 5.

Чтоб узнать какие ветки у нас есть в проекте можно ввести команду
git branch

По стандарту у нас всегда есть основная ветка main или master
На против ветки в которой мы находимся стоит звёздочка *
Для создание новой ветки нужно ввести команду
git branch имя_ветки
*также можно создать новую ветку и сразу переключиться на неё с помощью команду
git checkout -b имя_ветки


Чтоб переключиться на новую ветку нужно использовать команду
git checkout имя_ветки

В случае успеха выведется надпись
Switched to branch 'имя_ветки'

Теперь если мы изменим файл, и добавим его с помощью git add 
Изменения будут зафиксированы на данной ветке
Чтоб закинуть ветку на удалённый репозиторий нужно ввести следующую команду
git push -u origin имя_ветки

Можно увидеть, что у нас в репозитории теперь две ветки.
Мы можем смотреть историю коммитов в новой ветке 
git log имя_ветки --oneline

Теперь попробуем слить две ветки в одну - для начала нужно перейти на ветку в которую будем сливать.
В нашем случае это main
git checkout main

Теперь можно увидеть, что изменения из файлов пропали.
Сольём наши ветки использовав следующую команду
git merge new-branch

Увидим, что наши изменения появились. 
Теперь рассмотрим - что делать если конфликты появились.
Конфликты возникают, когда изменения вносятся в один и тот же файл.

Для этого нам нужно найти конфликтующие данные - посмотреть, на суть конфликта и удалить лишнее.
А затем коммитим эти изменения.
