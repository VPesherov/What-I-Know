Для начала давайте вспомним, что такое API
API - (интерфейс программирования приложений, интерфейс прикладного программирования) (англ. application programming interface, API)
- набор готовых классов, процедур, функций, структур и констант, предоставляемых приложением (библиотекой, сервисом) или 
операционной системой для использования во внешних программных продуктах. Используется программистами при написании всевозможных
приложений.
Для чего оно вообще нужно? По сути просто для того, чтоб два разных приложения - одно клиентское, второе серверное - могли
с друг другом общаться.
На рисунке 6 описана Логика работы Web API. Каждый клиент должен уметь одинаково общаться с сервером.

Теперь разберём два параметры.
Аутенфикация и авторизация - их нужно уметь различать.
Аутенфикация - это подтверждение вашей личности, то есть что вы это вы.
Авторизация же - это проверка прав на какое либо действие, то есть авторизованы ли вы, что бы сделать что-то?
То есть это процесс выдачи прав на выполнение определённых действий.

И для пользования API - нужно авторизироваться.
