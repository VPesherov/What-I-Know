У джанги есть концепция создания проекта

Она гласит что проект должен состоять из нескольких приложений
По умолчанию подключённые приложения можно увидеть в settings.py в переменной INSTALLED_APPS
Для создание собственного приложения используем команду

1)python manage.py startapp <название приложения>
и создаётся папка приложения

2)В созданной папке лежат файлы

migrations - папка для работы с моделями и бд

__init__ - обозначение что папка это пакетный модуль

admin - для работы с админкой

apps - в файлике apps есть переменная name с названием нашего приложения, это имя используется для подключения приложения

tests - для юнит тестов

views - файл отвечающий за логику данного приложения
