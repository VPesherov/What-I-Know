Глобально есть два вида процессинга
1. CPU-bound - операции, которые требуют исключительно процессорного времени
2. I/O-bound - операции, которые привязаны к медленным устройствам (диски, сеть и тд)

Для CPU-bound используется вытесняющая многозадачность (Windows, Linux)

Для I/O-bound используется кооперативная многозадачность (когда вызов к I/O не блокирует вызывающий поток,
а по завершению оповещает его об окончании). Потоки помогают друг другу!

CPU-bound могут требовать параллелизацию и тогда нужно реально использовать многоядерность.

На картинке к теме 3 можно узнать какие библиотеке с чем работают.
